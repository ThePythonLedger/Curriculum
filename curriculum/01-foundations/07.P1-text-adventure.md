## 🧠 Conceptual Overview: State & Control Flow

Before writing code, we must understand how a game engine tracks reality under the hood. 

### 1. The Concept of "State"
In programming, **state** is the stored configuration of a system at a specific moment in time. Think of it as your program's short-term memory. In Python, we represent state using variables. 


When a player makes a choice, the program does two things:
1. It evaluates the choice using logical branches.
2. It modifies the **state** (e.g., subtracting from `health` or switching `has_token` from `False` to `True`).

### 2. Deep Nesting and Execution Contexts
When you nest an `if` statement inside another `if` statement, you are creating a conditional hierarchy:

```python
if choice1 == "left":
    # --- Context level 1 ---
    if choice2 == "right":
        # --- Context level 2 ---
        print("You found the exit!")

```
Python evaluates the inner conditional block **only** if the outer condition evaluates to True. If the outer condition is False, the interpreter completely skips the inner block, saving memory and processing time.
